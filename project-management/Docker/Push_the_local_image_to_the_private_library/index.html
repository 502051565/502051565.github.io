# 1.将本地镜像推送到私有库

  • 1.1.什么是Docker Registry
  • 1.2.将本地镜像推送到私有库案例

# 1.1.什么是Docker Registry

1 官方Docker Hub地址：https://hub.docker.com/ (opens new window)，中国大陆访问太慢了且准备被阿里云取代的趋势，不太主流。

2 Dockerhub、阿里云这样的公共镜像仓库可能不太方便，涉及机密的公司不可能提供镜像给公网，所以需要创建一个本地私人仓库供给团队使用，基于公司内部项目构建镜像。

​ Docker Registry是官方提供的工具，可以用于构建私有镜像仓库

    -d:后台运行,-p 5000:5000:端口映射,/zzyyuse/myregistry/:容器卷目录

  1. 将新镜像zzyyubuntu:1.2修改符合私服规范的Tag

按照公式： docker tag 镜像:Tag Host:Port/Repository:Tag

自己host主机IP地址，填写你们自己的，不要粘贴错误，O(∩_∩)O

使用命令 docker tag 将zzyyubuntu:1.2 这个镜像修改为192.168.233.130:5000/zzyyubuntu:1.2

为啥要这样修改呢？因为是要符合私服规范

docker tag zzyyubuntu:1.2 192.168.233.130:5000/zzyyubuntu:1.2




 






[root@node2 ~]# docker tag  zzyyubuntu:1.2  192.168.233.130:5000/zzyyubuntu:1.2

  1. vim命令新增如下红色内容：vim /etc/docker/daemon.json
{
  "registry-mirrors": ["https://aa25jngu.mirror.aliyuncs.com"],
  "insecure-registries": ["192.168.233.130:5000"]
}

2
3
4

上述理由：docker默认不允许http方式推送镜像，通过配置选项来取消这个限制。====> 修改完后如果不生效，建议重启docker:systemctl restart docker

注意重启完docker之后，要重新启动私有服：docker run -d -p 5000:5000 -v /zzyyuse/myregistry/:/tmp/registry --privileged=true registry
